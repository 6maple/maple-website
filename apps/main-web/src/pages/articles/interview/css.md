# CSS 面经

## 盒模型

### 1. 核心构成（四个区域）

盒模型是浏览器对 HTML 元素进行布局的基础，任何元素都可以看作一个矩形盒子。

- **Content（内容区）**：放置文本或图片的实际区域。
- **Padding（内边距）**：内容与边框之间的透明区域。
- **Border（边框）**：包裹在内边距和内容外的线条。
- **Margin（外边距）**：盒子与其他元素之间的空白区域，用于控制外部间距。

### 2. 模型分类（计算差异）

通过 `box-sizing` 属性可以切换两种不同的空间计算逻辑：

- **标准盒模型（W3C 盒模型）**：
  - 设置方式：`box-sizing: content-box`（默认值）。
  - 计算公式：`实际宽度 = width + padding + border`。设置的 width 仅指内容区宽度，增加内边距或边框会导致盒子整体变大。
- **怪异盒模型（IE 盒模型）**：
  - 设置方式：`box-sizing: border-box`。
  - 计算公式：`实际宽度 = width`。设置的 width 已经包含了 padding 和 border，内部挤压内容区，盒子总尺寸保持不变。

## BFC

### 1. 核心定义

BFC 是 Web 页面的 CSS 渲染控制蓝图中的一个环境。可以将它理解为一个独立的容器，容器内部的子元素布局不会影响到外部，外部元素也不会影响到 BFC 内部。

### 2. 触发条件（如何创建 BFC）

在实际开发中，最常用的触发方式包括：

- **根元素**：即 `<html>`。
- **浮动元素**：`float` 值不为 `none`。
- **定位元素**：`position` 值为 `absolute` 或 `fixed`。
- **显示模式**：`display` 值为 `inline-block`、`flex`、`grid`、`table-cell` 等。
- **溢出处理**：`overflow` 值不为 `visible`（如 `hidden`、`auto`、`scroll`）。

### 3. 渲染规则（BFC 内部的表现）

- **垂直排列**：内部的 Box 会在垂直方向一个接一个地放置。
- **外边距重叠**：属于同一个 BFC 的两个相邻 Box 的上下 `margin` 会发生重叠。
- **空间占位**：每个元素的左外边距与包含块的左边界相接触（即使存在浮动）。
- **独立隔离**：BFC 的区域不会与 `float` box 重叠。
- **高度计算**：计算 BFC 的高度时，浮动元素也参与计算。

### 4. 实际应用场景

- **防止外边距重叠（Margin Collapse）**：
  当上下相邻元素 `margin` 合并导致布局异常时，可以将其中一个元素包裹在新的 BFC 容器中。
- **清除内部浮动（解决高度塌陷）**：
  父元素设置 `overflow: hidden` 触发 BFC，使其在计算高度时包含浮动的子元素，从而撑开父容器，替代传统的 `clearfix` 方案。
- **实现自适应两栏布局**：
  左侧元素 `float: left`，右侧主区域通过触发 BFC（如 `overflow: hidden`），利用“BFC 区域不与 float box 重叠”的规则，实现右侧宽度根据剩余空间自适应，且不会环绕到浮动元素下方。

## 响应式布局的实现方式

### 1. 核心基础方案（底层能力）

- **媒体查询 (Media Queries)**：响应式布局的基石。通过 `@media` 针对不同屏幕宽度（Breakpoints）编写差异化样式。通常采用“移动端优先”策略，先写基础样式，再通过 `min-width` 向上适配大屏。
- **百分比布局 (Percentage)**：最原始的流式布局方式。将子元素的宽高、内边距设置为父元素的百分比。缺点是计算关系复杂（基于父元素的不同属性），难以处理高度适配。
- **Flexbox (弹性盒布局)**：现代布局的首选方案。通过 `flex-wrap: wrap` 实现自动换行，配合 `flex-grow` 和 `flex-shrink` 动态分配剩余空间。适用于局部组件的排列与对齐。
- **Grid (网格布局)**：二维布局利器。相比 Flex 的轴线布局，Grid 能同时控制行和列。在处理复杂的页面大框架（如后台管理系统的侧边栏与内容区响应）时效率最高。

### 2. 现代单位方案（等比例缩放）

- **rem 适配方案**：
  - **原理**：rem 相对于 HTML 根元素的 `font-size`。
  - **实践**：配合 `lib-flexible` 或原生 JavaScript 动态计算根字号。将设计稿 px 转换为 rem，实现页面整体等比例缩放。
- **vw/vh 视口单位**：
  - **原理**：1vw 等于视口宽度的 1%。
  - **实践**：直接利用 CSS 原生单位，不再依赖 JS 计算。通过 `postcss-px-to-viewport` 等插件在构建阶段自动转换 px，是目前更推荐的无侵入式方案。

### 3. 组件化与工程化方案（高效率开发）

- **断点组件化 (Breakpoint Component)**：在 Vue 中封装 `v-responsive` 指令或响应式 Hook（如 `useWindowSize`）。根据当前窗口宽度条件渲染不同的组件（如移动端渲染抽屉导航，PC端渲染顶部菜单）。
- **响应式 UI 框架**：
  - **栅格系统 (Grid System)**：如 Element Plus 或 Ant Design Vue 中的 `Row/Col` 组件，通过 `xs`, `sm`, `md`, `lg`, `xl` 属性快速配置不同尺寸下的列数。
  - **原子化 CSS (Tailwind CSS)**：利用类名前缀（如 `md:flex-row`, `lg:text-xl`）在 HTML 层面快速构建响应式界面，大幅减少 CSS 代码量。
- **图片响应式**：
  - 使用 `<img>` 的 `srcset` 属性或 `<picture>` 标签，根据屏幕分辨率加载不同尺寸的图片资源，优化带宽与加载速度。

### 4. 实践思维总结

- **优先移动端**：先确保小屏体验，再逐步增加大屏细节。
- **组合拳策略**：大框架用 **Grid**，局部列表用 **Flex**，等比例缩放用 **vw/vh**，特殊差异点用 **Media Queries**。
- **内容驱动**：不要死守固定断点（如 768px），应根据内容在什么宽度下“变丑”来决定是否添加新的媒体查询。
