# 计算机网络面经

## 首屏加载优化

### 1. 资源体积压缩（减法）

- **路由懒加载与组件动态导入**：利用 Vite/Webpack 的 `import()` 分解首屏 JS 体积。确保只有首页代码在初始阶段加载，非首页资源在路由切换时按需获取。
- **依赖包按需引入**：针对 UI 组件库（如 Element Plus、Ant Design Vue），通过插件配置按需引入组件和样式，避免将整库打入 `vendor.js`。
- **Tree Shaking 优化**：编写代码时坚持使用 ES Modules 规范，确保构建工具能剔除未引用的死代码。
- **静态资源压缩**：
  - **图片优化**：使用 WebP 格式替代传统格式；对于小图标，采用 IconFont 或 SVG 替代图片。
  - **Gzip/Brotli 压缩**：在服务端（Nginx）和构建端开启高倍率压缩，大幅减少网络传输字节数。

### 2. 传输与响应速度（加速）

- **CDN 内容分发**：将第三方库（Vue、Axios 等）以及静态图片托管至 CDN。利用物理距离最近的节点加速响应，并利用浏览器对不同域名的并发下载特性。
- **HTTP/2 协议升级**：启用多路复用（Multiplexing），解决 HTTP/1.1 的头部阻塞问题，实现资源并行下载。
- **合理利用缓存**：
  - **强缓存与协商缓存**：对带有哈希值的打包文件设置强缓存，减少重复请求。
  - **预加载（Preload/Prefetch）**：通过 `<link rel="preload">` 提前加载关键字体或首页核心脚本。

### 3. 渲染架构与执行策略（效率）

- **SSR (服务端渲染) 或 SSG (静态生成)**：对于 SEO 敏感或首屏性能要求极高的项目，采用 Nuxt.js 等方案。直接由服务器返回填充好数据的 HTML，实现“秒开”效果。
- **骨架屏（Skeleton Screen）**：在数据异步加载期间，使用 CSS 绘制的占位骨架屏替代传统的 Loading 转圈。这虽不改变物理加载时间，但能极大地降低用户感知到的等待焦虑。
- **初始化逻辑优化**：
  - **非核心接口后置**：首页初始化只调用关键接口，非即时展现的数据（如埋点、配置信息）放在 `onMounted` 之后或 `requestIdleCallback` 中处理。
  - **减少阻塞**：将非必要的第三方插件脚本设置为 `defer` 或 `async` 加载。

### 4. 专项细节（极致）

- **CSS 抽取与内联**：将影响首屏布局的核心 CSS 提取并内联在 HTML 中，避免因加载外部 CSS 文件导致的“白屏”或“样式闪烁”。
- **字体裁剪**：若首页使用了特殊字体，通过工具仅保留首屏出现的字符，将几兆的字体文件减小到几十 KB。
